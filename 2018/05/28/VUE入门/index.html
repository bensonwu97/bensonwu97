2.4 路由对象

我们在定义map清单的时候，可以在任何一个路由上面都可以设置用户的参数。此时可以被路由对象拿到。
首先说Vue给router上面定义了一个beforeEach()的函数，这个函数会在路由发生变化的时候发生。
这个函数接受一个回调函数，回调函数接受一个系统给的transition对象。这个对象身上有to、from属性，表示你从哪个路径过来，去哪儿。有一个next()方法，表示方形请求，还有redirect()表示重定向。
通常用这个东西，检查登录状态。

在template中，可以用$route来自由枚举路由对象中的任何属性：

路由对象内置的属性，可以直接用在template里面也可以用在data里面，用this.$route来表示即可。
$route.path
类型: string
字符串，对应当前路由的路径，总是解析为绝对路径，如 “/foo/bar”。

$route.params
类型: Object
一个 key/value 对象，包含了 动态片段 和 全匹配片段，如果没有路由参数，就是一个空对象。

$route.query
类型: Object
一个 key/value 对象，表示 URL 查询参数。例如，对于路径 /foo?user=1，则有 $route.query.user == 1，如果没有查询参数，则是个空对象。

$route.hash
类型: string
当前路由的 hash 值 (带 #) ，如果没有 hash 值，则为空字符串。

$route.fullPath
类型: string
完成解析后的 URL，包含查询参数和 hash 的完整路径。

2.5 具名路径
map清单中，每一个路由都可以起一个名字，叫做name属性，此时在a标签中就能用name和params配合来跳转：
router.map({
‘/xuesheng/:xuehao’: {
“name” : “xuesheng”,
component: Xuesheng,
}
});

三、vue-loader

3.1 配置package.json和webpack.config.js文件

一提到loader就是webpack里面的增强功能，可以增加webpack的功能，webpack之所以受欢迎就是因为有很多loader。比如之前的babel-loader。
vue可以提供.vue文件，被编译成为js文件甚至css文件，需要配置webpack的vue-loader。
官网：https://github.com/vuejs/vue-loader
注意版本号：
NOTE: the master branch (9.0+) only works with Vue 2.x. For usage with Vue 1.x, see the 8.x branch.

核心目标就是编译.vue文件，.vue文件就是创造性的把style、template、script写在一个文件中的文件。开发的是偶更爽！不用去切换什么控制器、服务。乱八七糟的全在一个文件中，webpack帮我们分门别类的上页面。

我们把vue-loader作为项目的依赖
cnpm install vue-loader –save-dev

### 3.6 说说style区域
style区域的内容会自动加到html里面的head里面。也就是说，webpack中的bundle.js帮我们写了一个类似：
